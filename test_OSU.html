About the Course

This course teaches you to think like a data scientist—to follow claims back to evidence, spot common pitfalls, and communicate results responsibly. We’ll use friendly tools (JASP, Desmos, StatKey) and add lightweight coding options (R & Python) for students who want to go deeper. Our emphasis is **reasoning over software**: you’ll learn what a result means, not just how to click into one.

Format
Short demos · guided practice · discussion
Assessment
Low-stakes practice + short write-ups
Goal
Explain what the data supports—clearly

Course Topics (with the “why”)

Evidence & Arguments

  • Claims vs. evidence; operationalizing questions
  • Observational vs. experimental designs
  • Reproducibility and transparency

Why: avoid asserting more than data can support.

Distributions & Variation

  • Center, spread, shape; outliers
  • Visualization choices: when plots mislead

Why: “average” often hides important structure.

Inference Basics

  • Confidence intervals; p-values (what they are, what they aren’t)
  • t-tests, ANOVA, correlation in context

Why: quantify uncertainty; don’t overclaim.

Resampling Logic

  • Randomization tests; bootstrap intervals (StatKey demos)
  • Simulation to validate intuition

Why: distribution-free insight builds trust.

Modeling & Assumptions

  • Simple regression; residuals and diagnostics
  • Confounding, Simpson’s paradox, data leakage

Why: models help explain—but assumptions matter.

Ethics & Communication

  • Data provenance, consent, privacy
  • Plain-language reporting; visual honesty

Why: clarity and integrity are part of the grade.

Key Definitions (fast reference)

Confidence Interval (CI)
A range of plausible values for a population parameter, derived from the sample.
p-value
Probability of observing data as or more extreme than yours if the null were true.
Effect Size
Magnitude of a relationship or difference (e.g., Cohen’s d, r). Complements p-values.
Confounding
A hidden factor that influences both X and Y, distorting the relationship.
Randomization Test
Assess significance by shuffling labels to simulate the null world.
Bootstrap
Resample with replacement from the sample to estimate variability (e.g., CI).
Overfitting
Model memorizes noise; looks great on training data, fails on new data.
Reproducibility
Independent analysts can obtain the same results following your steps.

Recommended Workflows

JASP (No-code)

  1. Import CSV → Inspect columns & missingness
  2. Describe → Descriptives (plots + summary)
  3. Run test (t, ANOVA, corr) → Check assumptions
  4. Export APA table → Write 3-sentence interpretation

StatKey (Concept demo)

  1. Select procedure → choose example data
  2. Bootstrap or randomization → visualize sampling
  3. Capture CI/p-value → compare to JASP/Python

R (Reproducible)

  1. readr::read_csv() → dplyr clean
  2. ggplot2 plot → check assumptions
  3. t.test(), aov() or lm()
  4. Report effect size + CI in plain English

Python (Notebook)

  1. pandas.read_csv → .info(), .describe()
  2. matplotlib/seaborn plots
  3. scipy.stats or statsmodels for tests/models
  4. Narrate results + attach figure

FAQ

I’m new to stats. Will I be okay?

Yes. We emphasize concepts and communication, not heavy math. Tools are chosen to help you see what’s happening.

Do I have to code?

No. JASP/StatKey/Desmos can cover all essentials. R/Python are optional for students who want more control or reproducibility.

Which tool should I pick?

Start with JASP for analyses and StatKey for resampling intuition. Add R or Python if you want transparent, reproducible workflows or plan on data-heavy majors.

How do I install JASP / R / Python on my home computer?
  • JASP: Download & install
  • R + RStudio: Install R (CRAN), then RStudio
  • Python: Install Miniconda, then create an environment as shown above
What file formats do we use?

CSV for datasets; PDF or DOCX for short write-ups; JASP files (.jasp), R scripts (.R), or notebooks (.ipynb) if you choose coding tools.

How are assignments graded?

On clarity of reasoning and correct interpretation (including limits/assumptions), not just getting a particular number.
